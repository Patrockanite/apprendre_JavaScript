NameSpaces

Si l'on déclare une variable globale avec un même nom dans plusieurs bibliothèques, au fil du chargement, les variables vont s'écraser l'une après l'autre.

Palierau problème en simulant un nameSpace

On peurt palier facilement à ce problème en créant des objets littéraux et en appeleant leurs méthodes plutôt qu'en utilisant des fonctions

Exemples dans les fichiers JS

Immdetiatly Invoked Function Expression (IIFE) (prononcé IFI)

On peut invoquer une fonction de manière immédiate. Exemple dans le fichier IIFE.js

Isoler son code dans une IIFE

On peut se créer un pseudo NameSpace en encapuslant le code de "Mon_objet" dans une fonction IIFE (voir IIFE.JS). Si l'on veut appeler des méthodes depuis la Global Scope (window) il faudra créer une propriété (window.toto = Mon_objet ; en passant window en argument de la fonction IIFE. Ainsi window.toto et Mon_objet pointeront au même endroit de la mémoire.
